Each card connects the live stream to a real production use-case. Click the query to see the data behind it.
⛓
DeFi Block Monitoring Solana
A DeFi arbitrage bot watches slot progression to detect network congestion — each slot is a ~400 ms window. If slots stop arriving, the bot pauses trades. Run to see the most recent slots and timestamps, exactly as a trading system would consume them.
📡
Trending Topic Detection Bluesky
At 20 posts/sec, platforms detect trending topics using 30-second sliding windows. Language distribution is a proxy for geographic spread — a sudden spike in one language can signal a breaking news event in that region.
📝
Vandalism & Edit-War Detection Wikipedia
Wikipedia's anti-vandalism bot (ClueBot NG) consumes this same SSE stream. Signals: an editor making many tiny edits (low diff_bytes) in a short window, or many edits to the same article. Run to find high-frequency editors and their byte impact.
⚡
Kafka as a Rate Buffer All sources
Bluesky produces 20 events/sec but ClickHouse batch-inserts every 2 seconds. Without Kafka, the consumer must match producer speed exactly. Use the controls above to stop Bluesky, then restart it — watch lag spike in the Kafka Anatomy panel then drain as the consumer catches up.
🗓
Multi-Source Event Timeline All sources
News aggregators (Bloomberg Terminal, Reuters) ingest blockchain confirmations, social signals, and editorial changes into one unified store. ClickHouse's columnar engine makes this cross-source pivot query instant even at millions of rows.
🔍
Full-Text Search Indexing Bluesky
Search engines like Elasticsearch consume social streams to build near-real-time indexes. The raw post text arriving at 20/sec would typically go: Kafka → Flink tokeniser → Elasticsearch index. Run to see the actual post text flowing through right now.
